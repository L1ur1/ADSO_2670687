Concepto de GIT

GIT es un sistema de control de versiones distribuido, lo que significa que un clon local del proyecto es un repositorio de control de versiones completo. Estos repositorios locales plenamente funcionales permiten trabajar sin conexión o de forma remota con facilidad. La principal prioridad de Git es conservar cualquier código fuente en el que se esté trabajando o esté en gestión, tales aspectos como los archuvos, su contenido, los directorios, las veriones, las etiquetas y las confirmaciones.

PRINCIPALES COMANDOS DE GIT

  • GIT INIT: Este comando funciona para inicializar un proyecto con Git

  • GIT ADD: Permite agregar cambios al área de reparación

  • GIT CONFIG: Permite identificar al usuario con el nombre y el correo

  • GIT COMMIT: Permite realizar un comentario sobre los cambios realizados

  • GIT LOG: Permite explorar el historial del respositorio

  • GIT REMOTE: Es el que permite agregar automaticamente al enlace que genera GitHub

  • GIT STATUS: Permite mostrar el estado del directorio de trabajo y del área de entorno del ensayo

  • GIT BRANCH: Es el que permite crear, enumerar y eliminar ramas, así como cambiar su nombre

  • GIT CHECKOUT: Permite desplazar al usuario entre las ramas creadas

  • GIT PUSH Permite actualizar/subir el documento a la nube Git
